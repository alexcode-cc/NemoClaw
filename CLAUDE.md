# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

NVIDIA NemoClaw is an open-source reference stack for running OpenClaw always-on assistants inside OpenShell sandboxes with NVIDIA inference support. It provides declarative policy control over network egress, filesystem access, and privilege escalation across multiple inference backends (NVIDIA Endpoint API, local NIM, vLLM, Ollama).

## Prerequisites

- Node.js >= 22.0.0
- Python >= 3.11
- [uv](https://docs.astral.sh/uv/) for Python dependency management

## Build & Development Commands

```bash
# Initial setup
npm install                                    # Root deps + prek hook install
cd nemoclaw && npm install && npm run build     # TypeScript plugin
cd nemoclaw-blueprint && uv sync               # Python blueprint deps

# Build
cd nemoclaw && npm run build      # One-time compile
cd nemoclaw && npm run dev        # Watch mode

# Lint & format
make check                        # All checks via prek (linters + formatters + type checks)
make format                       # Auto-format all source

# Tests (Vitest)
npm test                          # All tests (cli + plugin projects)
npx vitest run --project cli test/cli.test.js                    # Single CLI test
npx vitest run --project plugin nemoclaw/src/commands/slash.test.ts  # Single plugin test
npx vitest run --coverage        # With coverage

# Docs
make docs                         # Build Sphinx/MyST docs
make docs-live                    # Serve with auto-rebuild
```

## Architecture

This is a polyglot monorepo with three layers:

| Layer | Path | Language | Purpose |
|-------|------|----------|---------|
| CLI entry point | `bin/nemoclaw.js` | Node.js (zero deps) | User-facing CLI with color output |
| TypeScript plugin | `nemoclaw/` | TypeScript | OpenClaw plugin API: slash commands, inference routing, workspace migration |
| Python blueprint | `nemoclaw-blueprint/` | Python 3.11+ | Sandbox orchestration, policy management via OpenShell CLI |

**Additional directories:** `test/` (root-level integration tests), `docs/` (Sphinx/MyST documentation), `scripts/` (automation).

### Key Architectural Concepts

- **Plugin system**: `nemoclaw/src/index.ts` registers slash commands (`/nemoclaw status|eject|onboard|help`) and an inference provider with the OpenClaw SDK.
- **Workspace migration**: `nemoclaw/src/commands/migration-state.ts` (largest module) handles agent workspace migration between host and sandbox, with snapshot bundles and rollback support.
- **Blueprint execution**: Python subprocess communicates via stdout protocol — `PROGRESS:<0-100>:<label>` for progress, `RUN_ID:<id>` for run identifier.
- **Policy management**: YAML-based network policies with presets in `nemoclaw-blueprint/policies/presets/` (PyPI, Docker Hub, Slack, Jira).
- **Inference routing**: Four profiles (`default`, `ncp`, `nim-local`, `vllm`) routed through OpenShell to backends.
- **Onboarding config**: Wizard saves to `~/.openclaw/nemoclaw.json`; plugin reads and transforms into provider registration.
- **Plugin state**: Persisted at `~/.nemoclaw/state/nemoclaw.json`.

### Vitest Configuration

Three test projects defined in `vitest.config.ts`:
- **cli**: `test/**/*.test.{js,ts}` — integration tests for `bin/nemoclaw.js`
- **plugin**: `nemoclaw/src/**/*.test.ts` — unit tests for the TypeScript plugin
- **e2e-brev**: `test/e2e/brev-e2e.test.js` — end-to-end tests, only runs when `BREV_API_TOKEN` is set

Coverage thresholds are enforced via `ci/coverage-threshold.json` (lines: 95%, functions: 98%, branches: 86%, statements: 95%).

## Conventions

### Commit Messages

遵循 AngularJS Git Commit Message Conventions，所有提交訊息使用**繁體中文**撰寫。

格式：
```
<type>(<scope>): <subject>

<body>

<footer>
```

**type（必填，需與 `commitlint.config.js` 一致）：**
- `feat`：新功能
- `fix`：修復錯誤
- `docs`：文件變更
- `refactor`：重構（非新功能、非修復）
- `test`：新增或修改測試
- `chore`：建置流程或輔助工具變更
- `perf`：效能改善
- `ci`：CI/CD 變更
- `merge`：合併提交

**scope（選填）：** 影響範圍，例如 `cli`、`blueprint`、`plugin`、`docs`。

**subject（必填）：** 簡短描述變更內容，使用繁體中文，不加句號。

**body（選填）：** 詳細說明變更動機與前後差異，使用繁體中文。

**footer（選填）：** 記錄 Breaking Changes 或關閉的 Issue。Breaking Changes 以 `BREAKING CHANGE:` 開頭。

範例：
```
feat(cli): 新增 --profile 旗標至 onboard 指令

支援使用者在 onboard 階段直接指定推論設定檔，
省去後續手動設定的步驟。

BREAKING CHANGE: onboard 指令的預設行為已變更，
不再自動選擇 default 設定檔。
```

```
fix(blueprint): 修正缺少 API 金鑰時的錯誤處理
```

```
docs: 更新快速入門指南以配合新版安裝精靈
```

### Git Hooks (prek)

Hooks are managed by prek (auto-installed via `npm install`). Key hooks:
- **pre-commit**: formatters, linters, plugin unit tests
- **commit-msg**: commitlint
- **pre-push**: `tsc --noEmit`, pyright

Run all hooks manually: `npx prek run --all-files`

### Documentation

- Edit pages in `docs/`, never in `.agents/skills/docs/` (autogenerated).
- After changing docs, regenerate skills: `python scripts/docs-to-skills.py docs/ .agents/skills/docs/ --prefix nemoclaw`
- Style: MyST Markdown, active voice, second person, present tense, no hedge words, no emoji in prose.
- Word casing: NemoClaw, OpenClaw, OpenShell, NVIDIA (always capitalized in prose).

### No External Project Links

Do not add links to third-party code repositories or unofficial resources in docs or code. Links to official tool documentation (Node.js, Python, uv) are acceptable.

### Code Style

- **TypeScript** (`nemoclaw/`): ESLint strict type-checked, Prettier (semicolons, double quotes, trailing commas, 100-char width, 2-space indent).
- **Python** (`nemoclaw-blueprint/`): Ruff for linting/formatting, Pyright strict mode, 100-char line length.
- **Dockerfiles**: Linted by Hadolint in CI.
- SPDX license headers required on all source files.
## Project Overview

NVIDIA NemoClaw is an open-source reference stack for running [OpenClaw](https://openclaw.ai) always-on assistants inside [NVIDIA OpenShell](https://github.com/NVIDIA/OpenShell) sandboxes with NVIDIA inference (Nemotron). It provides CLI tooling, a blueprint for sandbox orchestration, and security hardening.

**Status:** Alpha (March 2026+). Interfaces may change without notice.

## Architecture

| Path | Language | Purpose |
|------|----------|---------|

| `test/e2e/` | Bash/JS | End-to-end tests (Brev cloud instances) |
| `docs/` | Markdown (MyST) | User-facing docs (Sphinx) |
| `k8s/` | YAML | Kubernetes deployment manifests |

## Code Style and Conventions

### Commit Messages

Conventional Commits required. Enforced by commitlint via prek `commit-msg` hook.

```text
<type>(<scope>): <description>
```

Types: `feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `ci`, `perf`, `merge`

### SPDX Headers

Every source file must include an SPDX license header. The pre-commit hook auto-inserts them:

```javascript
// SPDX-FileCopyrightText: Copyright (c) 2026 NVIDIA CORPORATION & AFFILIATES. All rights reserved.
// SPDX-License-Identifier: Apache-2.0
```

For shell scripts use `#` comments. For Markdown use HTML comments.

### JavaScript

- `bin/` and `scripts/`: **CommonJS** (`require`/`module.exports`), Node.js 22.16+
- `test/`: **ESM** (`import`/`export`)
- ESLint config in `eslint.config.mjs`
- Cyclomatic complexity limit: 20 (ratcheting down to 15)
- Unused vars pattern: prefix with `_`

### TypeScript

- Plugin code in `nemoclaw/src/` with its own ESLint config
- CLI type-checking via `tsconfig.cli.json`
- Plugin type-checking via `nemoclaw/tsconfig.json`

### Shell Scripts

- ShellCheck enforced (`.shellcheckrc` at root)
- `shfmt` for formatting
- All scripts must have shebangs and be executable

### No External Project Links

Do not add links to third-party code repositories, community collections, or unofficial resources. Links to official tool documentation (Node.js, Python, uv) are acceptable.
