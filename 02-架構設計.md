# 架構設計

## 三層式架構

NemoClaw 採用三層式架構，每一層職責明確：

```
使用者
  │
  ▼
┌──────────────────────────────┐
│  CLI 入口點 (bin/nemoclaw.js) │  ← 零相依 Node.js，色彩輸出，命令分派
│  語言：JavaScript (ESM)       │
└──────────┬───────────────────┘
           │ 呼叫 openshell CLI / 管理本地狀態
           ▼
┌──────────────────────────────┐
│  TypeScript 插件 (nemoclaw/) │  ← OpenClaw 插件 API 實作
│  語言：TypeScript (ES2022)    │
│  職責：                       │
│  • Slash 指令處理             │
│  • 推論提供者註冊             │
│  • 工作區遷移狀態管理         │
└──────────┬───────────────────┘
           │ 透過 stdout 協定通訊
           ▼
┌──────────────────────────────┐
│  Python Blueprint            │  ← 沙箱編排、策略管理
│  (nemoclaw-blueprint/)       │
│  語言：Python 3.11+          │
│  職責：                       │
│  • 沙箱建立 / 銷毀           │
│  • 推論路由設定               │
│  • 遷移快照 / 還原            │
│  • 網路策略套用               │
└──────────┬───────────────────┘
           │ openshell CLI
           ▼
┌──────────────────────────────┐
│  OpenShell 沙箱              │  ← Docker 隔離環境
│  內含：OpenClaw + NemoClaw    │
│  安全機制：                   │
│  • Landlock 檔案系統隔離      │
│  • 網路出口白名單             │
│  • 非特權使用者 (sandbox)     │
└──────────────────────────────┘
```

## 元件互動

### 插件註冊流程

```
OpenClaw Host
    │
    │ 呼叫 register(api)
    ▼
nemoclaw/src/index.ts
    │
    ├── api.registerCommand("nemoclaw", handler)
    │       └── 分派至 slash.ts → status | eject | onboard | help
    │
    ├── loadOnboardConfig()
    │       └── 讀取 ~/.nemoclaw/config.json
    │
    └── api.registerProvider(providerPlugin)
            └── 包含模型目錄（Nemotron 系列）
                ├── id, label, contextWindow (131072)
                └── auth: Bearer + NVIDIA_API_KEY
```

```

    │   spawn("python", ["runner.py"]) │
    │ ───────────────────────────────► │
    │                                  │
    │   stdout: "RUN_ID:nc-20260323-a1b2c3d4"

    │ ◄─────────────────────────────── │
    │                                  │
```

### 狀態管理分工

| 狀態類型 | 儲存位置 | 管理者 | 用途 |
|----------|----------|--------|------|
| 插件狀態 | `~/.nemoclaw/state/nemoclaw.json` | `blueprint/state.ts` | lastAction, blueprintVersion, sandboxName |
| Onboard 設定 | `~/.nemoclaw/config.json` | `onboard/config.ts` | endpoint, model, provider, credentialEnv |
| 沙箱註冊表 | `~/.nemoclaw/sandboxes.json` | `bin/lib/registry.js` | 所有沙箱的 metadata、預設沙箱 |
| 憑證 | `~/.nemoclaw/credentials.json` | `bin/lib/credentials.js` | API key（環境變數優先） |
| 執行紀錄 | `~/.nemoclaw/state/runs/{id}/` | `runner.py` | 每次 apply 的 plan 快照 |
| 遷移快照 | `~/.nemoclaw/snapshots/{ts}/` | `migration-state.ts` / `snapshot.py` | Host 設定備份 |

## Docker 映像架構

```
Dockerfile（多階段建置）
│
├── 階段 1：builder
│   ├── node:22-slim
│   ├── npm install + tsc 編譯 TypeScript 插件
│   └── 輸出 nemoclaw/dist/
│
└── 階段 2：runtime
    ├── node:22-slim + python3 + curl + git
    ├── 建立 sandbox 使用者（UID/GID 隔離）
    ├── 安裝 OpenClaw 2026.3.23
    ├── 安裝 NemoClaw 插件
    ├── 分割 .openclaw 目錄：
    │   ├── /sandbox/.openclaw     ← 唯讀（root 擁有，chmod 444）
    │   └── /sandbox/.openclaw-data ← 可寫（symlink 回 .openclaw）
    ├── 以 Python 腳本產生 openclaw.json：
    │   ├── 從 os.environ 讀取 CHAT_UI_URL（防注入）
    │   ├── 產生唯一 auth token（secrets.token_hex(32)）
    │   └── 設定 CORS origins
    └── 鎖定 openclaw.json（DAC: root:root, 444）
```
