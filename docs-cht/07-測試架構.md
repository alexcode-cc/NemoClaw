# 測試架構

## 測試框架

- **框架：** Vitest 4.1.0
- **覆蓋率：** V8 provider
- **報告格式：** text + json-summary

## Vitest 設定（vitest.config.ts）

三個測試專案：

| 專案 | 包含路徑 | 語言 | 範圍 |
|------|----------|------|------|
| `cli` | `test/**/*.test.{js,ts}` + `src/lib/**/*.test.ts` | JavaScript + TypeScript | CLI 整合測試 + TypeScript 模組測試 |
| `plugin` | `nemoclaw/src/**/*.test.ts` | TypeScript | 插件單元測試 |
| `e2e-brev` | `test/e2e/brev-e2e.test.js` | JavaScript | Brev 短期實例 E2E 測試（需 `BREV_API_TOKEN`） |

覆蓋率門檻已拆分為兩個檔案：

| 門檻檔案 | lines | functions | branches | statements |
|----------|-------|-----------|----------|------------|
| `ci/coverage-threshold-cli.json` | 22% | 32% | 22% | 22% |
| `ci/coverage-threshold-plugin.json` | 95% | 98% | 86% | 95% |

CLI 門檻較低反映 CJS → TypeScript 遷移的過渡狀態，將隨遷移完成逐步提高。

## 執行指令

```bash
# 所有測試
npm test

# 指定專案
npx vitest run --project cli
npx vitest run --project plugin

# 單一測試檔
npx vitest run --project cli test/cli.test.js
npx vitest run --project plugin nemoclaw/src/commands/slash.test.ts

# 覆蓋率
npx vitest run --coverage

# 覆蓋率閾值檢查（CI 使用，TypeScript 版本）
npx tsx scripts/check-coverage-ratchet.ts
```

## CI/CD 流程

### GitHub Actions 工作流程

| 工作流程 | 觸發 | 說明 |
|----------|------|------|
| `pr.yaml` | Pull Request | 基本檢查（lint、test、build） |
| `main.yaml` | main 合併 | 主分支檢查（新增於 e9d3120） |
| `sandbox-images-and-e2e.yaml` | PR + main | 沙箱映像建置與 E2E（從 pr.yaml 拆出） |
| `commit-lint.yaml` | PR events | commitlint + Squash merge PR 標題驗證 |
| `dco-check.yaml` | PR | DCO 簽署檢查（新增於 88bb222） |
| `nightly-e2e.yaml` | 每日 UTC 00:00 | 含 GPU E2E、失敗時建立 Issue |
| `e2e-brev.yaml` | 手動/CI 呼叫 | Brev CPU 實例 E2E（GCP） |
| `base-image.yaml` | main 合併（Dockerfile.base 變更） | 基礎映像推送 GHCR |
| `docs-preview-pr.yaml` | PR | 文件嚴格建置 + HTML artifact |
| `docs-preview-deploy.yaml` | workflow_run | 文件預覽部署 |
| `docker-pin-check.yaml` | 每週一 UTC 09:00 | Dockerfile 釘選摘要新鮮度 |
| `pr-limit.yaml` | PR | PR 數量速率限制 |

## 覆蓋率管理

使用 `scripts/check-coverage-ratchet.ts`（TypeScript 版本，取代原 bash 腳本）強制執行最低覆蓋率門檻。

CI 中每次 PR 分別檢查 CLI 與 plugin 覆蓋率是否低於各自門檻。
