# 架構設計

## 三層式架構

NemoClaw 採用三層式架構，每一層職責明確：

```
使用者
  │
  ▼
┌──────────────────────────────┐
│  CLI 入口點 (bin/nemoclaw.js) │  ← CommonJS，色彩輸出，命令分派
│  語言：JavaScript (CJS)       │     漸進遷移至 TypeScript (src/lib/)
│  新增：第三方軟體接受確認     │
└──────────┬───────────────────┘
           │ 呼叫 openshell CLI / 管理本地狀態
           ▼
┌──────────────────────────────┐
│  TypeScript 插件 (nemoclaw/) │  ← OpenClaw 插件 API 實作
│  語言：TypeScript (ES2022)    │
│  職責：                       │
│  • Slash 指令處理             │
│  • 推論提供者註冊             │
│  • 工作區遷移狀態管理         │
└──────────┬───────────────────┘
           │ 透過 stdout 協定通訊
           ▼
┌──────────────────────────────┐
│  Blueprint 靜態資產 +         │  ← 策略定義（YAML）
│  TypeScript 編排器            │     編排邏輯已整合至插件層
│  (nemoclaw-blueprint/ +      │
│   nemoclaw/src/blueprint/)   │
│  職責：                       │
│  • 沙箱建立 / 銷毀           │
│  • 推論路由設定               │
│  • 遷移快照 / 還原            │
│  • 網路策略定義               │
│  • SSRF 端點驗證              │
└──────────┬───────────────────┘
           │ openshell CLI
           ▼
┌──────────────────────────────┐
│  OpenShell 沙箱              │  ← Docker 隔離環境
│  內含：OpenClaw + NemoClaw    │
│  安全機制：                   │
│  • Landlock 檔案系統隔離      │
│  • 網路出口白名單             │
│  • 非特權使用者 (sandbox)     │
│  • 閘道程序隔離 (gateway)     │
│  • 沙箱跨閘道重啟存活        │
└──────────────────────────────┘
```

## 元件互動

### 插件註冊流程

```
OpenClaw Host
    │
    │ 呼叫 register(api)
    ▼
nemoclaw/src/index.ts
    │
    ├── api.registerCommand("nemoclaw", handler)
    │       └── 分派至 slash.ts → status | eject | onboard | help
    │
    ├── loadOnboardConfig()
    │       └── 讀取 ~/.nemoclaw/config.json
    │
    └── api.registerProvider(providerPlugin)
            └── 包含模型目錄（Nemotron 系列）
                ├── id, label, contextWindow (131072)
                └── auth: Bearer + 對應 API KEY
```

### 狀態管理分工

| 狀態類型 | 儲存位置 | 管理者 | 用途 |
|----------|----------|--------|------|
| 插件狀態 | `~/.nemoclaw/state/nemoclaw.json` | `blueprint/state.ts` | lastAction, blueprintVersion, sandboxName |
| Onboard 設定 | `~/.nemoclaw/config.json` | `onboard/config.ts` | endpoint, model, provider, credentialEnv |
| 沙箱註冊表 | `~/.nemoclaw/sandboxes.json` | `bin/lib/registry.js` | 所有沙箱的 metadata、預設沙箱（原子寫入 + 檔案鎖定） |
| 憑證 | `~/.nemoclaw/credentials.json` | `bin/lib/credentials.js` | API key（環境變數優先、不安全 HOME 回退拒絕） |
| 第三方軟體接受 | `~/.nemoclaw/usage-notice.json` | `src/lib/usage-notice.ts` | 接受版本與時間戳 |
| 執行紀錄 | `~/.nemoclaw/state/runs/{id}/` | `blueprint/runner.ts` | 每次 apply 的 plan 快照 |
| 遷移快照 | `~/.nemoclaw/snapshots/{ts}/` | `migration-state.ts` / `blueprint/snapshot.ts` | Host 設定備份 |

## Docker 映像架構

```
Docker 映像架構（雙層設計）

Dockerfile.base（基礎映像，推送至 GHCR）
├── node:22-slim@sha256:... （釘選摘要）
├── apt 套件（精確版本釘選）
├── gosu 1.19（SHA-256 校驗，多架構）
├── 使用者/群組：gateway (UID 999)、sandbox (UID 1000)
├── .openclaw 目錄結構與 symlink（含 .openclaw/memory）
├── OpenClaw CLI（釘選版本）
└── PyYAML（釘選版本）

Dockerfile（PR 特定層）
│
├── 階段 1：builder
│   ├── node:22-slim
│   ├── npm ci + tsc 編譯 TypeScript 插件 + CLI 模組
│   └── 輸出 nemoclaw/dist/ + src/lib/ 編譯產出
│
└── 階段 2：runtime（FROM ${BASE_IMAGE}）
    ├── 安裝 NemoClaw 插件（npm ci --omit=dev）
    ├── Blueprint 本地解析
    ├── Brave Search 設定（若有 BRAVE_API_KEY）
    ├── 安全強化：
    │   ├── 移除 gcc/g++/make/netcat
    │   ├── ulimit -u 512（防 fork bomb）
    │   ├── capsh 權限降級
    │   ├── iptables（網路策略強制執行）
    │   ├── PATH 鎖定
    │   └── CHAT_UI_URL 烘焙至映像
    ├── 閘道程序隔離（gateway 使用者）
    ├── 設定雜湊釘選（sha256sum）
    ├── .openclaw symlink immutable
    └── 鎖定 openclaw.json（DAC: root:root, 444）
```
