# NemoClaw 沙箱操作指南

本文件詳細說明在 NemoClaw 沙箱內的操作方式，涵蓋檔案存取、設定修改、OpenClaw 運行、網路策略管理與外部服務連接。

## 沙箱檔案系統

### 目錄結構總覽

```
/sandbox/                              ← 沙箱使用者 Home 目錄（可寫）
├── .openclaw/                         ← OpenClaw 設定根目錄
│   ├── openclaw.json                  ← 核心設定（唯讀，root:root, chmod 444）
│   ├── agents → .openclaw-data/agents
│   ├── extensions → .openclaw-data/extensions
│   ├── workspace → .openclaw-data/workspace
│   ├── skills → .openclaw-data/skills
│   ├── hooks → .openclaw-data/hooks
│   ├── identity → .openclaw-data/identity
│   ├── devices → .openclaw-data/devices
│   ├── canvas → .openclaw-data/canvas
│   ├── cron → .openclaw-data/cron
│   └── update-check.json → .openclaw-data/update-check.json
│
├── .openclaw-data/                    ← 可寫狀態目錄（所有 symlink 指向此處）
│   ├── agents/main/agent/             ← Agent 執行狀態、auth-profiles.json
│   ├── extensions/                    ← 已載入的擴充套件
│   ├── workspace/                     ← Agent 工作區（文件、程式碼）
│   ├── skills/                        ← 可用技能清單
│   ├── hooks/                         ← 自訂 Hooks
│   ├── identity/                      ← 裝置/使用者身份
│   ├── devices/                       ← 已配對裝置
│   ├── canvas/                        ← Canvas 協作狀態
│   └── cron/                          ← 排程任務
│
├── .nemoclaw/                         ← NemoClaw 插件狀態（可寫）
│   ├── config.json                    ← Onboard 設定
│   ├── state/nemoclaw.json            ← 插件狀態
│   └── blueprints/0.1.0/             ← Blueprint 設定
│
└── (其他使用者建立的檔案和目錄)
```

### 唯讀 vs 可寫區域

| 路徑 | 權限 | 說明 |
|------|------|------|
| `/sandbox/.openclaw/openclaw.json` | 唯讀（root:root, 444） | 核心設定：auth token、CORS、模型路由。由 Landlock 強制保護 |
| `/sandbox/.openclaw/` 目錄本身 | 唯讀（Landlock） | 防止 Agent 竄改認證設定 |
| `/sandbox/.openclaw-data/` | 可寫 | Agent 狀態、工作區、技能、Hooks 等 |
| `/sandbox/.nemoclaw/` | 可寫 | NemoClaw 插件狀態與設定 |
| `/sandbox/` | 可寫 | 使用者 Home 目錄 |
| `/tmp/` | 可寫 | 暫存檔案 |
| `/usr/`, `/lib/`, `/etc/` | 唯讀 | 系統目錄 |

## 連線與基本操作

### 連線至沙箱

```bash
# 從 Host 端連線
nemoclaw my-assistant connect

# 連線後提示符變為：
sandbox@my-assistant:~$
```

### 在沙箱內執行單一指令（不進入互動 Shell）

```bash
# 從 Host 端直接執行沙箱內的指令
openshell sandbox connect my-assistant -- ls -la ~/.openclaw/
openshell sandbox connect my-assistant -- cat ~/.nemoclaw/config.json
```

### 離開沙箱

```bash
# 在沙箱內
exit
# 或按 Ctrl+D
```

## OpenClaw 操作

### 互動式對話（TUI）

```bash
# 在沙箱內啟動 TUI 介面
openclaw tui
```

TUI 提供互動式聊天介面，適合日常對話。按 `Ctrl+C` 離開。

### 單次訊息（CLI）

```bash
# 發送單一訊息並取得回應
openclaw agent --agent main --local -m "你好，請自我介紹" --session-id test

# 使用不同的 session（對話歷史獨立）
openclaw agent --agent main --local -m "寫一個 Python hello world" --session-id coding-01
```

`--session-id` 決定對話歷史的隔離範圍，相同 ID 的訊息共享對話脈絡。

### 查看已安裝插件

```bash
openclaw plugins list
```

### 查看裝置配對狀態

```bash
openclaw devices list --json
```

## 檔案存取與編輯

### 從 Host 端傳輸檔案

沙箱內無法使用 `apt install` 安裝工具，建議從 Host 端操作。

**上傳檔案至沙箱：**

```bash
# DEST 必須是目標目錄（以 / 結尾），非完整檔案路徑
openshell sandbox upload my-assistant /path/to/local-file.txt /sandbox/

# 上傳至工作區
openshell sandbox upload my-assistant IDENTITY.md /sandbox/.openclaw/workspace/

# 上傳整個目錄
openshell sandbox upload my-assistant ./my-skills/ /sandbox/.openclaw-data/skills/
```

> **注意：** `upload` 的目標路徑（DEST）必須是**目錄**，檔案會以原始檔名放入該目錄並覆蓋同名檔案。

**從沙箱下載檔案：**

```bash
# 下載單一檔案
openshell sandbox download my-assistant /sandbox/.nemoclaw/config.json /tmp/

# 下載整個工作區
openshell sandbox download my-assistant /sandbox/.openclaw/workspace/ /tmp/workspace-backup/
```

**編輯沙箱內檔案的推薦流程：**

```bash
# 1. 下載
openshell sandbox download my-assistant /sandbox/.openclaw/workspace/IDENTITY.md /tmp/

# 2. 在 Host 端編輯
vim /tmp/IDENTITY.md

# 3. 上傳回沙箱
openshell sandbox upload my-assistant /tmp/IDENTITY.md /sandbox/.openclaw/workspace/
```

### 在沙箱內直接編輯

```bash
nemoclaw my-assistant connect

# 使用內建的 vi 或 nano（若可用）
vi ~/.openclaw-data/workspace/IDENTITY.md
nano ~/.openclaw-data/workspace/IDENTITY.md

# 使用 heredoc 覆寫小檔案
cat > ~/.openclaw-data/workspace/notes.md <<'EOF'
# 我的筆記
這是在沙箱內建立的檔案。
EOF

# 使用 sed 進行行內替換
sed -i 's/舊文字/新文字/g' ~/.openclaw-data/workspace/IDENTITY.md
```

## OpenClaw 設定修改

### openclaw.json（核心設定 — 唯讀）

`/sandbox/.openclaw/openclaw.json` 是在建置時產生的核心設定，包含：

- 模型路由（`agents.defaults.model.primary`）
- 推論提供者設定（`models.providers`）
- 閘道設定（`gateway.mode`, `gateway.auth.token`）
- CORS 允許的來源（`gateway.controlUi.allowedOrigins`）

**此檔案無法在沙箱內修改**（root 擁有、chmod 444、Landlock 保護）。

若需變更模型或推論設定，從 Host 端操作：

```bash
# 變更推論路由（Host 端）
openshell inference set --provider ollama-local --model 新模型名稱

# 或重新執行 Onboard
NEMOCLAW_RECREATE_SANDBOX=1 \
NEMOCLAW_MODEL=新模型名稱 \
nemoclaw onboard
```

### NemoClaw 設定（可修改）

`/sandbox/.nemoclaw/config.json` 記錄 Onboard 設定，可在沙箱內修改：

```bash
# 在沙箱內查看
cat ~/.nemoclaw/config.json

# 從 Host 端修改
openshell sandbox download my-assistant /sandbox/.nemoclaw/config.json /tmp/
vim /tmp/config.json
openshell sandbox upload my-assistant /tmp/config.json /sandbox/.nemoclaw/
```

設定結構：

```json
{
  "endpointType": "custom",
  "endpointUrl": "https://inference.local/v1",
  "model": "nemotron-3-super:cloud",
  "provider": "ollama-local",
  "providerLabel": "Local Ollama",
  "credentialEnv": "OPENAI_API_KEY",
  "profile": "inference-local",
  "onboardedAt": "2026-03-24T..."
}
```

### Agent 工作區身份設定

Agent 的身份文件位於工作區，可自由修改：

```bash
# 上傳自訂身份文件
cat > /tmp/IDENTITY.md <<'EOF'
# Agent 身份

你是一位專業的軟體工程助理，擅長 Python 和 TypeScript。
請使用繁體中文回答問題。
EOF

openshell sandbox upload my-assistant /tmp/IDENTITY.md /sandbox/.openclaw/workspace/
```

## 技能與 Hooks 管理

### 新增自訂技能（Skills）

技能以 Markdown 檔案定義，放置於 `/sandbox/.openclaw-data/skills/`：

```bash
# 建立技能目錄
mkdir -p /tmp/my-skill/

# 建立技能定義
cat > /tmp/my-skill/SKILL.md <<'EOF'
---
name: my-custom-skill
description: 我的自訂技能
instructions: 當使用者詢問特定主題時使用此技能
---

## 使用方式

這是一個自訂技能的範例內容。
Agent 會在適當時機參考此技能的知識。
EOF

# 上傳至沙箱
openshell sandbox upload my-assistant /tmp/my-skill/ /sandbox/.openclaw-data/skills/
```

### 新增自訂 Hooks

Hooks 在 OpenClaw 啟動時載入，用於擴充行為：

```bash
# 建立 Hook 目錄
mkdir -p /tmp/my-hook/

# 建立 Hook 定義
cat > /tmp/my-hook/HOOK.md <<'EOF'
---
name: my-hook
description: 自訂 Hook 範例
---
EOF

# 建立 Handler
cat > /tmp/my-hook/handler.js <<'EOF'
export default async function myHook() {
  // Hook 邏輯
}
EOF

# 上傳至沙箱
openshell sandbox upload my-assistant /tmp/my-hook/ /sandbox/.openclaw-data/hooks/
```

| NVIDIA 推論 | integrate.api.nvidia.com | 模型推論 |

### 新增網路策略預設集

```bash
# 互動式選擇（從 Host 端）
nemoclaw my-assistant policy-add

# 列出已套用與可用的策略
nemoclaw my-assistant policy-list
```

可用的預設集（支援以數字序號選擇）：

| 預設集 | 用途 | 允許的端點 |
|--------|------|------------|
| `brave` | Brave Search | api.search.brave.com |
| `pypi` | pip install | pypi.org, files.pythonhosted.org |
| `npm` | npm install | registry.npmjs.org, registry.yarnpkg.com |
| `docker` | Docker pull | registry-1.docker.io, nvcr.io |
| `huggingface` | HF Hub | huggingface.co, cdn-lfs.huggingface.co |
| `slack` | Slack API | slack.com, api.slack.com, hooks.slack.com |
| `telegram` | Bot API | api.telegram.org |
| `discord` | Discord API | discord.com, gateway.discord.gg |
| `jira` | Jira API | *.atlassian.net, api.atlassian.com |
| `outlook` | MS Graph | graph.microsoft.com, outlook.office365.com |

### 動態新增自訂端點（運行時）

無需重建沙箱，直接套用策略檔案：

```bash
# 建立自訂策略 YAML
cat > /tmp/custom-policy.yaml <<'EOF'
version: 1
network_policies:
  my_api:
    name: my_api
    endpoints:
      - host: api.example.com
        port: 443
        protocol: rest
        enforcement: enforce
        tls: terminate
        rules:
          - allow:
              method: "*"
              path: "/**"
EOF

# 套用至運行中的沙箱（Host 端）
openshell policy set /tmp/custom-policy.yaml
```

> **注意：** 動態變更僅在當前 session 有效，沙箱停止後重置為基礎策略。永久變更需修改 `nemoclaw-blueprint/policies/openclaw-sandbox.yaml` 並重新 onboard。

### 即時核准網路請求（TUI）

當 Agent 嘗試存取不在策略中的端點時，OpenShell 會攔截請求。你可以在 Host 端開啟 TUI 即時審核：

```bash
# 開啟 OpenShell 監控 TUI
openshell term
```

TUI 會顯示：
- 沙箱狀態與推論提供者
- 即時網路活動
- 被攔截的請求（含目標 Host、Port、發起的二進位檔、HTTP 方法/路徑）

對每個攔截的請求，可以選擇：
- **Approve**：核准該端點存取（僅限當前 session）
- **Deny**：保持封鎖

## 外部服務連接

### Telegram Bot 整合

在 Host 端（非沙箱內）啟動 Telegram Bridge：

```bash
# 設定環境變數
export TELEGRAM_BOT_TOKEN="your-bot-token"    # 從 @BotFather 取得
export NVIDIA_API_KEY="your-api-key"           # 若使用 NVIDIA 推論
export SANDBOX_NAME="my-assistant"
export ALLOWED_CHAT_IDS="123456789"            # 選用：限制存取的 Chat ID

# 啟動 Bridge
node scripts/telegram-bridge.js

# 或使用 NemoClaw CLI
nemoclaw start
```

Bridge 的運作方式：
1. 輪詢 Telegram API 取得新訊息
2. 透過 SSH 連入沙箱執行 `openclaw agent`
3. 擷取 Agent 回應並回傳至 Telegram

支援的 Telegram 指令：
- `/start` — 顯示 Bot 介紹
- `/reset` — 清除對話歷史
- 任何其他文字 → 送至 Agent 處理

### Dashboard（瀏覽器介面）

NemoClaw 安裝時已自動設定 Dashboard 埠轉發：

```bash
# 確認埠轉發運行中
openshell forward start --background 18789 my-assistant

# 在瀏覽器開啟
# http://127.0.0.1:18789/
```

若需取得帶認證 token 的 URL：

```bash
# 從沙箱內取得 auth token
openshell sandbox connect my-assistant -- \
  python3 -c "import json; print(json.load(open('/sandbox/.openclaw/openclaw.json'))['gateway']['auth']['token'])"

# 完整 URL 格式：http://127.0.0.1:18789/#token=<token>
```

## 常見操作範例

### 變更 Agent 使用的模型

```bash
# 從 Host 端變更推論路由（支援多種提供者）
openshell inference set --provider ollama-local --model 新模型名稱
openshell inference set --provider compatible-endpoint --model 新模型名稱

# 驗證沙箱推論狀態（Host 端）
nemoclaw my-assistant status

# 驗證（在沙箱內）
openclaw agent --agent main --local -m "你使用的是什麼模型？" --session-id test
```

### 備份沙箱工作區

```bash
# 從 Host 端下載整個工作區
openshell sandbox download my-assistant /sandbox/.openclaw/workspace/ ~/nemoclaw-backup/

# 還原
openshell sandbox upload my-assistant ~/nemoclaw-backup/workspace/ /sandbox/.openclaw/
```

### 在沙箱內安裝 Python 套件

需先套用 pypi 策略預設集：

```bash
# Host 端：確認 pypi 策略已套用
nemoclaw my-assistant policy-list

# 若未套用
nemoclaw my-assistant policy-add
# 選擇 pypi

# 沙箱內：安裝套件
nemoclaw my-assistant connect
pip install requests --user
```

### 在沙箱內使用 Git

GitHub 存取在基礎策略中已預先允許：

```bash
# 在沙箱內
nemoclaw my-assistant connect
git clone https://github.com/user/repo.git
cd repo
# ... 操作 ...
```

### 重啟 OpenClaw 閘道（沙箱內）

```bash
# 在沙箱內
pkill -f "openclaw gateway" || true
nohup openclaw gateway run > /tmp/gateway.log 2>&1 &
```

## 沙箱跨閘道重啟存活

NemoClaw 支援沙箱在閘道重啟後自動恢復（2d29a02）。閘道升級或意外重啟不會中斷沙箱作業。

### 主機重啟後重連

```bash
# 1. 確認 Docker 運行中
docker info

# 2. 檢查沙箱狀態
openshell sandbox list

# 3. 重啟閘道（若未自動恢復）
openshell gateway start

# 4. 連線至沙箱
nemoclaw my-assistant connect

# 5. 重啟輔助服務（若有使用）
nemoclaw start
```

## 疑難排解

### 常見問題

| 問題 | 解決方式 |
|------|----------|
| `nemoclaw` 指令找不到 | 檢查 PATH 設定，執行 `source ~/.bashrc` 或重新開啟終端 |
| Node.js 版本太舊 | 需 22.16+，安裝器會嘗試透過 nvm 升級 |
| 映像建置 OOM | 需至少 8 GB RAM，或啟用 swap（自動/手動） |
| 沙箱 DNS 無法解析 | Colima 需 CoreDNS 修補，WSL2 會自動跳過 |
| 無效的沙箱名稱 | 需符合 RFC 1123（小寫英數字、連字號，最多 63 字元） |
| Onboard 閘道啟動失敗 | 自動指數退避重試 |
| 推論逾時 | 使用 `nemoclaw <name> status` 檢查推論設定 |
| 被攔截的網路請求 | 使用 `openshell term` 即時審核，或新增策略預設集 |
| WSL2 Docker 路由異常 | 自動跳過 OLLAMA_HOST=0.0.0.0 設定 |
| DGX Spark cgroup v2 | 使用 `nemoclaw setup-spark` |
